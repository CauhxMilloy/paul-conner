Twilight Horizon Entertainment is a small indie video game development team that I started. I started it in 2009 with a few friends. We've been working on games in our spare time. Sadly, since we're all either full time students and/or have full time jobs, we havn't been able to release a game. This is very much a labor of love sort of thing. However, it has given me a lot of experience in game design. We have a strong focus in documentation and prototyping. Since time is in such sort supply, we make sure not to waste any time implementing something that we're unsure of.

Working with Twilight Horizon has given me a lot of opportunities to write libraries for making games. I have extensive C# libraries (we've used XNA a lot for games we've worked on) and I've been able to re-use those libraries in school projects. These libraries cover a multitude of game engine necessities, including: Object Network Serialization, Graphics APIs, Platforming, HLSL Shaders and Job Thread Pools; there's a lot to be had.

Along side those libraries, I've made quite a few developer tools. One of the tools I'm most proud of is the custom art/animation pipeline that I've made. My goal was for the artists that I work with to import art assets from any program they may use and be able to - through a few steps - import that as animated sprites (that automagically work in engine). This required making my own PSD parser (so they can load PSD files, and turn on/off layers on the fly in my dev tools). I fully realize that many tools out there aren't (for the lack of better descriptive words) "good". I'd like to use my programming knowledge to make sure the people I work with aren't frustrated with bad tools. Recently, I was working on porting some of my libraries to Objective-C, so my art pipeline would also work for iPhone development as well.

I'm also working on my own game engine in C++. The scope of this engine will be large, to encompass any game we could think of making. I'd also like it to work on multiple platforms, and be well abstracted so all high-level game code would not (necessarily) be too tightly coupled with any platform.
